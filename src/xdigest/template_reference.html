Artigo: "How to Build an Agent"

@thorstenball at 13 abr, 14:18 + @bradyjeong at 13 abr, 14:48 · agents, tutorial, amp

ampcode.com/notes/how-to-build-an-agent

There's no secret behind coding agents. It's an LLM, a loop, and enough tokens. The article builds a fully functional code-editing agent in under 400 lines of Go, step by step:

  • Basic chatbot with Anthropic SDK (90 lines)
  • Tool use via API: read_file, list_files, edit_file — the model returns JSON describing the tool call, you execute, return the result
  • The agentic loop: LLM requests tool → execute → return result → repeat until done
  • System prompt that shapes behavior

The modern tool use API is just a structured version of "wink if you want me to raise my arm." The real complexity in Claude Code, Codex, Amp, Pi is polish, not architecture.

Worth reading in full. 30 min to understand the foundation of every coding agent.

Podcast: Latent Space × Stay SaaSy

@latentspacepod at 13 abr, 15:56 · management, AI adoption, token budgets

YouTube, 57 min — "The best engineers don't write the most code. They delete the most code."

Core thesis: 2026 is the year token budgets become the new headcount budget. The shift from fixed pricing to consumption will create the biggest management challenge of the decade.

  • Token budgets as individual cost (4:00): Imagine if each employee's laptop cost between $500 and $50,000/year depending on usage. An OpenAI engineer spends ~1 billion tokens/day (~$2.5M/year). Fully loaded cost of an engineer will include token budget as a primary component.
  • Build vs Buy recalibrated (19:00): AI makes building cheaper, but mature software is still more complex than it looks. Heuristic: if you can replace it with a spreadsheet, build it. If not, think twice.
  • Automate executives, not juniors (29:00): 99% of leadership decisions are "knowable" and standardizable. The top of the org tree may be more automatable than the leaves.
  • HITL burnout (44:00): Running 10 simultaneous agents is viable but exhausting. When a critical PR arrives at end of day, the reviewer is cognitively depleted. Real risk to reliability.
  • Amazon went down 6h from vibe-coded change (44:00): If Amazon can't maintain code review culture under AI shipping pressure, nobody is immune.
  • Future of PRs (49:00): Either review goes 100% automated, or the industry will need something like pilot fatigue limits before reviewing critical code.
